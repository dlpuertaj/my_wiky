# BRAIN — LLM-Maintained Personal Knowledge Wiki

A persistent, compounding knowledge base where an LLM agent (Claude Code) reads
your raw sources and incrementally builds a structured, interlinked wiki of
markdown files. You curate sources and ask questions. The LLM does all the
writing, cross-referencing, and maintenance. Obsidian is the viewer.

## Why This Instead of RAG?

Most LLM + document workflows (NotebookLM, ChatGPT file uploads, RAG systems)
re-derive knowledge from scratch on every question. Nothing accumulates. Ask a
question that requires synthesizing five documents, and the LLM has to find and
piece together the fragments every time.

This is different. When you add a source, the LLM reads it, extracts key
information, and **integrates it into an existing wiki** — updating entity pages,
revising summaries, flagging contradictions, strengthening the evolving
synthesis. The knowledge is compiled once and kept current, not re-derived on
every query. The wiki gets richer with every source you add and every question
you ask.

## Architecture

Three layers:

```
BRAIN/
├── CLAUDE.md                  # Schema — tells the LLM how to maintain the wiki
├── README.md                  # This file
│
├── raw/                       # Layer 1: Raw Sources (immutable)
│   ├── paper.pdf
│   ├── paper.pdf.extracted.md # (generated by extract.py)
│   ├── article.html
│   ├── notes.txt
│   └── diagram.png
│
├── wiki/                      # Layer 2: The Wiki (LLM-owned)
│   ├── index.md               # Catalog of all pages
│   ├── log.md                 # Chronological activity log
│   ├── sources/               # One summary per ingested source
│   ├── entities/              # People, orgs, tools, places, products
│   ├── concepts/              # Ideas, frameworks, theories, methods
│   └── maps/                  # Synthesis, comparisons, overviews
│
├── outputs/                   # Layer 3: Query Results
│   └── comparison-x-vs-y.md
│
└── scripts/
    └── extract.py             # Text extractor for PDF, HTML, web URLs
```

**Raw sources** — Your curated collection of source documents. Articles, papers,
images, data files. Immutable — the LLM reads from them but never modifies them.
This is your source of truth.

**The wiki** — A directory of LLM-generated markdown files. Summaries, entity
pages, concept pages, comparisons, an overview, a synthesis. The LLM owns this
layer entirely. It creates pages, updates them when new sources arrive, maintains
cross-references, and keeps everything consistent. You read it; the LLM writes it.

**The schema** (`CLAUDE.md`) — Tells the LLM how the wiki is structured, what
the conventions are, and what workflows to follow. This is the key configuration
file. You and the LLM co-evolve it over time as you figure out what works for
your domain.

## Prerequisites

Install Python dependencies for the text extractor:

```bash
pip install pdfplumber beautifulsoup4 requests
```

## Usage

### Setup

1. Open this directory in Obsidian as a vault.
2. Open Claude Code in a terminal pointed at this directory.
3. You now have the LLM on one side and Obsidian on the other. The LLM makes
   edits; you browse the results in real time.

### Step 1: Ingest a Source

Drop a file into `raw/` and tell Claude to process it.

**For text and markdown files:**
```
> I added raw/my-notes.txt — please ingest it.
```

**For PDFs or HTML files** (extract text first):
```
> Run python scripts/extract.py raw/paper.pdf, then ingest it.
```

**For web articles:**
```
> Run python scripts/extract.py https://example.com/article, then ingest it.
```

**For images** (screenshots, diagrams, photos):
```
> I added raw/diagram.png — please ingest it.
```
Claude reads images directly using vision — no extraction needed.

**What happens during ingest:**
1. Claude reads the full source.
2. Discusses key takeaways with you and asks what to emphasize.
3. Creates a source summary page in `wiki/sources/`.
4. Creates or updates entity pages in `wiki/entities/`.
5. Creates or updates concept pages in `wiki/concepts/`.
6. Cross-references with existing pages, flags contradictions.
7. Updates `wiki/index.md` and appends to `wiki/log.md`.

A single source can touch 10-15 wiki pages.

### Step 2: Ask Questions (Query)

Ask anything against the accumulated wiki:

```
> What are the main differences between X and Y?
> Summarize everything we know about [entity].
> What contradictions exist in the sources about [topic]?
```

Claude reads the wiki index, finds relevant pages, and synthesizes an answer
with citations. Substantial answers (comparisons, analyses) are saved to
`outputs/` and can be filed back into the wiki as map pages — so your
explorations compound in the knowledge base just like ingested sources.

### Step 3: Maintain the Wiki (Lint)

Periodically ask Claude to health-check the wiki:

```
> Please lint the wiki.
```

Claude will:
- Identify orphan pages with no inbound links.
- Find concepts mentioned but lacking their own page.
- Flag contradictions between pages.
- Flag stale claims superseded by newer sources.
- Suggest missing cross-references.
- Suggest new sources to investigate for gaps.

## Tips

- **Ingest one at a time** and stay involved. Read the summaries, check the
  updates, guide what to emphasize. Quality compounds faster than quantity.
- **File good answers back into the wiki.** A comparison you asked for, an
  analysis, a connection you discovered — these shouldn't disappear into chat
  history. Ask Claude to save them as map pages.
- **Browse the Obsidian graph view** as the wiki grows. The link structure
  reveals how your knowledge connects.
- **Co-evolve the schema.** If a workflow isn't working, edit `CLAUDE.md`
  together with Claude. The schema is a living document.
- **Use the log.** `wiki/log.md` gives you a timeline of everything that's
  happened. Useful for resuming after a break.

## Supported Source Formats

| Format | Method |
|---|---|
| `.txt`, `.md` | Read directly |
| `.pdf` | `python scripts/extract.py raw/file.pdf` |
| `.html` | `python scripts/extract.py raw/file.html` |
| Web URL | `python scripts/extract.py https://...` |
| `.png`, `.jpg`, `.jpeg`, `.webp` | Read directly (Claude vision) |
